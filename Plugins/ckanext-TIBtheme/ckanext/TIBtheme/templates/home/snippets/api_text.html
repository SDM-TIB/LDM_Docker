{{ "APIs angeboten vom Leibniz-Datenmanager" }}

Der Leibniz Data Manager (LDM) ist eine Open-Source- und frei verfügbare webbasierte Anwendung für Forschungsdatenmanagement (FDM). Um die Integration mit externen Werkzeugen und Arbeitsabläufen zu erleichtern, bietet der LDM eine Reihe von APIs an, die verschiedene Zwecke im Bereich des Datenmanagements erfüllen.

Die angebotenen APIs können in den folgenden Bereichen eingesetzt werden:

  • Service für Datenvergleich: Diese API ermöglicht den direkten Vergleich von CSV-Datensätzen nebeneinander. Laden Sie zwei oder mehr CSV-Dateien hoch, verbinden Sie diese über eine gemeinsame Schlüsselspalte und erkunden Sie das Ergebnis in einer filterbaren Tabelle. Visualisierungen können als Linien-, Streu- oder Balkendiagramm mit optionaler logarithmischer Skala erstellt werden.
    • Schrittweises Laden von Ressourcen – beginnen Sie mit einer einzelnen CSV-Datei und ergänzen Sie weitere
    • Automatischer Left-Join über die erste gemeinsame Spalte
    • Filterbare Tabelle mit spaltenbasierter Suche und Seitennavigation (DataTables)
    • Interaktive Diagramme mit wählbaren Serien und Filterung auf sichtbare Zeilen (Plotly)
    • Verfügbar als Docker-Container oder lokale Python-Anwendung
    Diese API ist verfügbar auf GitHub.
  • Service für die Ausführung von Jupyter Notebooks: Eine containerisierte, multiuserfähige JupyterHub-Instanz mit einer einheitlichen REST-API zur Verwaltung von Notebooks. Entwickelt für die Integration in externe Anwendungen, die temporäre Jupyter-Umgebungen starten und Notebooks für Endnutzer bereitstellen müssen.
    • Pool konfigurierbarer Gastnutzer für gleichzeitige Sitzungen (z. B. guest0..guestN)
    • Automatische Bereinigung inaktiver Server mit konfigurierbaren Zeitlimits und Kernel-Lebenszyklusverwaltung
    • Pro-Container-Ressourcenlimits für CPU, Arbeitsspeicher und Festplattenplatz
    • Admin-Dashboard zur Konfiguration von Limits und Überwachung aktiver Sitzungen
    • Einheitliche REST-API über einen einzigen Port für alle Operationen
    Diese API ist verfügbar auf GitHub.
  • Service zur Erstellung von Knowledge Graphs: Diese API transformiert heterogene Datenquellen mithilfe von SDM-RDFizer und RML-Mappings in RDF-Knowledge-Graphs. RML (RDF Mapping Language) ist eine deklarative Sprache, mit der definiert wird, wie Datenelemente aus Quellen wie CSV, JSON oder XML auf RDF-Subjekte, Prädikate und Objekte abgebildet werden. Die API ermöglicht es Nutzern, Datensätze und RML-Mappingregeln einzureichen, den Transformationsprozess automatisch auszuführen und den resultierenden Knowledge Graph in einen angegebenen Ausgabeordner zu exportieren.
    • Unterstützt heterogene Quellformate (CSV, JSON, XML, relationale Datenbanken)
    • Automatisierte RDF-Tripel-Generierung auf Basis von RML-Mappingregeln
    • Ausgabe des Knowledge Graphs in einen konfigurierbaren Ordner
    Diese API ist verfügbar auf GitHub.
  • Service zur Integration von Knowledge Graphs: Diese API bildet Metadateneigenschaften von Datensätzen auf diejenigen ab, die in einer Mapping-Vorlage definiert sind. Der Zweck dieses Services besteht darin, Metadaten von Datensätzen auf die Struktur abzubilden, die von der Datensatzvorlage des Leibniz Data Managers (LDM) verlangt wird, um den Import von Datensätzen in die Plattform zu erleichtern.
    • Nutzerdefinierte Mapping-Vorlagen zur Verknüpfung vorhandener Metadateneigenschaften
    • Unterstützung für verschachtelte Metadatenstrukturen über Iterator-Konfiguration
    • Erzeugung einer strukturierten Ausgabedatei, die mit dem LDM-Template kompatibel ist
    Diese API ist verfügbar auf GitHub.
  • Service zur Exploration von Knowledge Graphs: Diese API liefert Metadaten zu Datensätzen, die mit einer wissenschaftlichen Publikation oder mit der ORCID eines Autors verbunden sind. Sie unterstützt sowohl die Abfrage einzelner Einträge als auch die Stapelverarbeitung mehrerer DOIs in einem einzigen Aufruf.
    • Abfrage von Datensatzmetadaten anhand einer DOI (einzeln oder als Batch mehrerer DOIs)
    • Abfrage von Datensatzmetadaten anhand der ORCID eines Autors
    • Abruf von Datensätzen, die Werke eines Autors zitieren, anhand der ORCID
    • Unterstützt sowohl GET- als auch POST-Anfragen
    Diese API ist verfügbar auf GitHub.

Alle verfügbaren APIs finden Sie in diesem GitHub-Repository.

Bitte kontaktieren Sie uns unter service.ldm@tib.eu, wenn Sie Kommentare oder Fragen haben.

{{ "APIs offered by the Leibniz Data Manager" }}

The Leibniz Data Manager (LDM) is an open-source and free web-based application for Research Data Management (RDM). To facilitate integration with external tools and workflows, the LDM offers a series of APIs that fulfil various data management purposes.

The APIs offered can be applied in the following areas:

  • Service for Data Comparison: This API enables direct side-by-side comparison of CSV datasets. Upload two or more CSV files, join them on a shared key column, and explore the result in a filterable table. Visualisations can be rendered as line, scatter, or bar charts with an optional logarithmic scale.
    • Incremental resource loading — start with a single CSV and join additional files one at a time
    • Automatic left-join on the first shared column across datasets
    • Filterable table with column-level search and pagination (DataTables)
    • Interactive charts with selectable series and filtering to currently visible rows (Plotly)
    • Available as a Docker container or a local Python application
    This API is available on GitHub.
  • Service for Jupyter Notebook Execution: A containerised, multi-user JupyterHub instance with a unified REST API for notebook management. Designed for integration into external applications that need to spawn temporary Jupyter environments and serve notebooks to end users.
    • Configurable pool of guest users for concurrent sessions (e.g. guest0..guestN)
    • Automatic cleanup of idle servers with configurable timeouts and kernel lifecycle management
    • Per-container resource limits for CPU, memory, and disk
    • Admin dashboard for configuring limits and monitoring active sessions
    • Unified REST API on a single port for all operations
    This API is available on GitHub.
  • Service for Knowledge Graph Creation: This API transforms heterogeneous data sources into RDF knowledge graphs using SDM-RDFizer executed with RML mappings. RML (RDF Mapping Language) is a declarative language for defining how data elements from sources such as CSV, JSON, or XML correspond to RDF subjects, predicates, and objects. The API allows users to submit datasets and RML mapping rules, automatically execute the transformation process, and output the resulting knowledge graph to a specified folder.
    • Supports heterogeneous source formats (CSV, JSON, XML, relational databases)
    • Automated RDF triple generation based on RML mapping rules
    • Knowledge graph output to a configurable folder
    This API is available on GitHub.
  • Service for Knowledge Graph Integration: This API maps dataset metadata properties to those defined in a mapping template. The purpose of this service is to map dataset metadata to the structure required by the dataset template of the Leibniz Data Manager (LDM), facilitating easier import of datasets into the platform.
    • User-defined mapping templates for linking existing metadata properties
    • Support for nested metadata structures via iterator configuration
    • Generates a structured output file compatible with the LDM dataset template
    This API is available on GitHub.
  • Service for Knowledge Graph Exploration: This API returns metadata about datasets associated with a scientific publication or an author's ORCID. It supports both single-entry lookups and batch processing of multiple DOIs in a single request.
    • Retrieve dataset metadata by DOI — individually or as a batch of multiple DOIs
    • Retrieve dataset metadata by an author's ORCID
    • Retrieve datasets citing an author's work, identified by ORCID
    • Supports both GET and POST requests
    This API is available on GitHub.

All available APIs can be found in this GitHub repository.

Please contact us at service.ldm@tib.eu if you have any comments or questions.
